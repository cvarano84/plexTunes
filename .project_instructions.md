# Plex Jukebox - Project Instructions

## Overview
Touch-optimized jukebox UI for Plex music libraries. Dark theme, widescreen touch monitor, NO vertical scrolling on main jukebox (mobile page is exception). No emojis. Text readable from across room using `clamp()`. Touch-primary (hover states don't work on touch - use `active:` instead).

## Tech Stack
- Next.js 14, TypeScript, Tailwind CSS, Prisma (Postgres), Framer Motion
- Audio: Web Audio API (3-band + 10-band EQ, analyser node for visualizations)
- Fonts: font-display class for headings

## Key Architecture
- `/app/jukebox/` - Main jukebox SPA (all views rendered inside jukebox-shell.tsx)
- `/app/mobile/` - Mobile remote page (vertical scrolling OK, phone-optimized)
- `/app/setup/` - Plex server configuration wizard
- `/lib/player-context.tsx` - Audio playback, EQ (simple 3-band + advanced 10-band), queue management
- Views: stations, artists, search, now-playing, artist-detail, album-detail, queue, settings, stats

## Database Models (Prisma)
- CachedArtist (with `summary` field for bio), CachedAlbum, CachedTrack
- Station, LibrarySyncStatus, PlexConfig, Settings
- NowPlayingState (for mobile remote state sync)
- RemoteAction (for mobile remote queue additions)

## EQ System
- 3-band (simple) + 10-band (advanced) always in audio chain
- source -> bass -> mid -> treble -> [10 peaking filters] -> analyser -> destination
- Inactive mode's gains zeroed. Mode switch doesn't reconnect nodes.
- 10-band: vertical sliders, old-school AV rack style, green/red fill bars

## Mobile Remote System
- Main jukebox publishes state to `/api/remote/state` every 5s
- Main jukebox polls `/api/remote/queue` every 5s for new tracks
- Mobile page polls `/api/remote/state` every 4s
- Mobile can only add to queue (no playback control, no station changes)
- QR code popup in player bar (phone icon), 15-second auto-hide

## Artist Detail View
- Full width layout: bio/photo top-left, album carousel middle-left, similar artists bottom-left, track list right panel (40% width)
- Bio fetched from `/api/artists/[id]/bio` (DB cache -> Plex summary -> LLM fallback)
- Similar artists from `/api/artists/[id]/similar` (Last.fm -> LLM -> same-genre fallback)

## Billboard Data
- Uses LLM knowledge (AI-estimated), NOT a live feed or dataset
- `json_object` response format with -1 convention for unknown values

## Settings View
- Providers: popularity (Deezer/Last.fm/Spotify), lyrics (LRCLIB/Genius)
- Metadata Dashboard: library counts, fill rates (popularity, summaries, billboard, artist bios), scan triggers
- Station management, display/behavior settings, Plex config
- Cover Art Fill %: per-page sliders (stationFillPct, artistFillPct, mixFillPct) — default 70
- Old mixArtistIconSize field kept in schema but unused in UI

## Night Mode
- Moon button in player bar (next to Sweet Fades), with popover for toggle + darkness slider
- Activation: saves current EQ state, applies bass-cut (bass: -8), kills all WLED panels via POST /api/wled/nightmode
- Deactivation: restores previous EQ, turns WLED panels back on
- Darkness overlay: fixed div with adjustable opacity (0-90%), z-index 40, pointer-events-none
- LED equalizer hidden when night mode is on
- State managed in player-context.tsx (nightMode, nightDim, setNightMode, setNightDim)

## Lyrics Caching
- CachedTrack has syncedLyrics, plainLyrics, lyricsCheckedAt fields
- /api/lyrics: cache-first with background synced lyrics search
- Frontend passes trackId param for caching

## Track Deduplication
- Artist detail: dedup popular tracks by title (keep highest popularity)
- Station tracks: dedupeTracks() helper applied at all return points

## External Services
- LASTFM_API_KEY: Not configured but code has fallbacks
- Deezer: Free, no key needed (default popularity provider)
- LRCLIB: Free, no key needed (default lyrics provider)
