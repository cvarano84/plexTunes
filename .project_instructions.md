# Plex Jukebox - Project Instructions

## Overview
A premium TouchTunes-style jukebox web app for Plex music libraries. Dark-themed, touch-optimized interface for large screens.

## Tech Stack
- Next.js 14, React 18, TypeScript, Tailwind CSS
- PostgreSQL via Prisma ORM
- External APIs: Plex (music library/streaming), Genius (lyrics), Spotify (popularity data)

## Design Language
- **Theme**: Dark, always-dark (no light mode). CSS variables set for dark jukebox aesthetic.
- **Colors**: Primary = purple (270 80% 60%), Accent = cyan (195 90% 50%), Background = deep navy (230 25% 7%)
- **Effects**: Neon text glow, album art glow shadows, vinyl spin animation, lyrics scroll mask
- **Touch**: Large click targets, touch-friendly, optimized for large screens
- **Fonts**: DM Sans (body), Plus Jakarta Sans (display), JetBrains Mono (mono)

## Architecture
- `/setup` - First-time Plex credential entry + library sync
- `/jukebox` - Main SPA-style interface with nav tabs: Stations, Artists, Search, Now Playing, Queue
- Client-side `PlayerProvider` context manages audio playback state
- All Plex communication proxied through API routes to keep token server-side

## Database Models
- `PlexConfig` - Stored Plex server URL and token
- `CachedArtist/Album/Track` - Full library cache from Plex
- `Station` - Auto-generated stations by decade+genre
- `LibrarySyncStatus` - Sync progress tracking

## API Routes
- `/api/plex/config` - GET/POST Plex credentials
- `/api/plex/library` - GET music library sections
- `/api/plex/sync` - POST to sync library, GET for progress
- `/api/plex/stream` - Proxy audio streaming
- `/api/plex/image` - Proxy album art images
- `/api/artists` - Paginated artist list with search
- `/api/artists/[id]` - Artist detail with albums/tracks
- `/api/albums/[id]` - Album detail with tracks
- `/api/stations` - List active stations
- `/api/stations/[id]/tracks` - Get shuffled station tracks
- `/api/search` - Global search across artists/albums/tracks
- `/api/lyrics` - Genius lyrics lookup
- `/api/popularity` - Batch Spotify popularity updates

## Key Decisions
- Always dark theme (class="dark" on html)
- Station generation requires 5+ matching tracks to be visible
- Spotify popularity cached in DB to avoid repeated API calls
- Images proxied through server to keep Plex token secure
- Docker files stored in public/ for easy download

## External API Secrets
- GENIUS_ACCESS_TOKEN - Genius lyrics API
- SPOTIFY_CLIENT_ID / SPOTIFY_CLIENT_SECRET - Spotify client credentials flow
- LASTFM_API_KEY - Last.fm popularity data (free)
- Deezer - No API key needed (free public search API)

## Provider System
- **Popularity**: Deezer (default #1, free), Last.fm (#2, needs API key), Spotify (#3, needs Premium)
- **Lyrics**: LRCLIB (default #1, free, synced timestamps), Genius (#2, needs token, plain text)
- Provider order/enable stored in localStorage: popProviderOrder, popProviderDisabled, lyricsProviderOrder, lyricsProviderDisabled
- Settings page has full provider management: reorder, enable/disable, test, top tracks viewer
- `/api/providers/test` - POST to test individual providers
- `/api/providers/top-tracks` - GET top tracks by popularity with distribution stats

## UI/UX Design Decisions
- **Horizontal swiping**: Stations grouped by decade, horizontally scrollable with album art mosaic cards
- **No emojis**: Station cards use album art from their track pool as visual identifiers
- **Combined Now Playing + Queue**: Side-by-side layout for widescreen monitors
- **Technics 1200 vinyl**: Turntable visualization with tonearm, grooves, and spinning record
- **LED Equalizer**: Canvas-based LED bar display at bottom of Now Playing (configurable bands & colors)
- **Infinite playback**: Stations auto-fetch more tracks when queue nears end
- **Idle timeout**: Configurable auto-switch to Now Playing view
- **Touch keyboard**: Theme-matched on-screen keyboard for all text inputs
- **Settings page**: Diagnostics (Plex, Genius, Spotify status), library stats, EQ config, idle timeout

## Billboard Chart Data
- LLM API (`gpt-4.1-mini`) used to look up Billboard Hot 100 peak position and weeks on chart
- Cached in `CachedTrack` fields: `billboardPeak`, `billboardWeeks`, `billboardCheckedAt`
- Songs >6 months old get permanent cache; newer songs refresh weekly
- `/api/tracks/billboard` - GET endpoint with `trackId` param
- Displayed in Now Playing view as colored badges (amber for peak, sky for weeks)

## Stats Page
- `/api/tracks/stats` - GET with `period` (30d/90d/1y/all) and `limit` params
- `stats-view.tsx` shows top played tracks with rank, play bars, billboard data
- Actions: Play All, Add All, individual Queue and Play Next buttons

## Custom Layout System
- `compact-queue` preset: [42, 43, 15] column splits
- `custom:L,M,R` format for user-defined splits via TripleSlider in settings
- Dual-handle drag slider component for adjusting 3-column layout ratios
